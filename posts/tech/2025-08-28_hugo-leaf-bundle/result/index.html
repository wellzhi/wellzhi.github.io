Leaf Bundle（叶子包），这是 Hugo 里用得最多的一种 Page Bundle，非常适合 PaperMod 文章。

📦 什么是 Leaf Bundle（叶子包）#

在 Hugo 中，一个 Page Bundle 就是“一个页面 + 绑定的资源文件”。

  • Leaf Bundle（叶子包）：指的是 目录里有 index.md 的内容包，它只能生成 一个页面。
  • 所有放在这个目录里的文件（图片、子 Markdown、附件）都属于这个页面的资源。

可以理解为： 👉 index.md = 文章正文 👉 同目录下的其他文件 = 这个文章的附属资源

📂 Leaf Bundle 的目录结构#

例如一篇文章 my-post：

content/posts/my-post/        ← 这是 Leaf Bundle
├── index.md                  ← 主文档（唯一页面）
├── part1.md                  ← 子文档（资源）
├── part2.md
├── image1.png
└── notes.pdf
  • index.md：这是文章的主体，Hugo 会渲染它生成 HTML 页面。
  • part1.md / part2.md：不会单独生成页面，它们是 index.md 的 资源文件，可以被 include。
  • image1.png / notes.pdf：同样是资源，可以用 .Resources.Get 获取。

⚙️ Shortcode 示例：resourceinclude.html#

你需要自己写一个 shortcode 来包含子 markdown。

在 layouts/shortcodes/resourceinclude.html 写入：

{{ $file := .Get 0 }}
{{ with .Page.Resources.GetMatch $file }}
  {{ .Content }}
{{ else }}
  <p style="color:red;">资源未找到：{{ $file }}</p>
{{ end }}

解释：

  • .Page.Resources.GetMatch → 在当前 index.md 所在的 Leaf Bundle 中查找资源。
  • .Content → 会渲染 Markdown 成 HTML（区别于 readFile）。
  • 如果没找到，就提示错误。

🔑 Leaf Bundle 的特点#

✅ 优点

  • 文章与资源绑定在一个目录里，移动文件不会出错。
  • Hugo 原生支持，调用资源时不用写复杂路径。
  • 图片、附件、子 md 都可以统一管理。

⚠️ 限制

  • 一个 Leaf Bundle 目录只能生成 一个页面（就是 index.md）。
  • 其他 md 文件不会独立渲染，只能作为资源被引用。

📌 使用场景#

  • 长文拆分：文章太长，可以拆成多个 part，最后在 index.md 中拼接。
  • 文档带插图：图片直接放在文章目录里，引用时只写文件名即可。
  • 附属资源：比如 PDF、代码片段、表格等，统一跟文章放在一起。

👉 总结一句： Leaf Bundle = 一篇文章 + 打包的所有资源文件，特别适合 PaperMod 的单篇文章场景。
