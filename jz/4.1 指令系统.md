# 指令构成

- 操作码 + 地址码 （可能有0-4个）

# 指令划分

- 按地址码数目分类
  - 零地址指令
  - 一地址指令
  - 二地址指令
  - 三地址指令
  - 四地址指令
- 按长度分类（区分：指令字长、机器字长、存储字长）
  - 定长指令字结构
  - 变长指令字结构
- 按操作码长度分类
  - 定长操作码
  - 变长操作码
- 按操作类型分类
  - 数据传送类：CPU、主存之间的数据传送
  - 运算类：算术逻辑操作、移位操作
  - 程序控制类：改变程序执行流
  - 输入输出类：CPU、IO设备之间的数据传送

# 扩展操作码指令格式

- 定长指令字结构：指令系统中所有指令的长度都相等
- 变长指令字结构：指令系统中各种指令的长度不相等
- 定长操作码：指令系统中所有指令的操作码长度相同

  - 优点：n位操作码字段的指令系统最大能够表示2^n条指令
    - 定长操作码对于简化计算机硬件设计，提高指令译码和识别速度很有利
  - 缺点：
    - 指令数量增加时会占用更多的固定位，留给表示操作数的地址的位数受限
- 可变长操作码：指令系统中各个指令的操作码长度可变

  - 优点：在指令字长有限的前提下仍保持比较丰富的指令种类
  - 缺点：增加了指令译码和分析的难度，使控制器的设计复杂化

举例：扩展操作码指令格式（定长指令字结构 + 可变长操作码）

- 定长指令字长16位，每个地址码占4位： 前4位为基本操作码OR + 3个4位地址字段 A1\A2\A3
- 高4位全部用于三地址指令
  - 0000***~1111***，有16条
    - 0000***~1110***，有15条（三地址指令）。1111留作扩展操作码，用于二地址指令
    - 0000 0000** ~ 1111 1110**，有15条（二地址指令）；1111 1111留作扩展操作码，用于一地址指令
    - 0000 0000 0000 * ~ 1111 1111 1110 *，有15条（一地址指令）；1111 1111 1111留作扩展操作码，用于零地址指令
    - 0000 0000 0000 0000 ~ 1111 1111 1110 1111，零地址指令共有16条

## 设计扩展操作码指令格式时，注意事项

- 不允许短码是长码的前缀（哈夫曼树”前缀编码“）
- 各指令的操作码一定不能重复

通常情况下，对于使用频率较高的指令，分配较短的操作码； 使用频率较低的指令，分配较长的操作码，从而尽可能减少指令译码和分析时间。

# 指令寻址

- 什么是指令寻址：确定下一条要执行的指令的存放地址，由程序计数器PC指明

每一条指令的执行都分为”取指令“和”执行指令“两个阶段

## 指令寻址方式

- 顺序寻址
  - （PC）+ "1" -> PC，"1"指一个指令字长
  - 每次取指令结束后，PC一定会+"1"
- 跳跃寻址
  - 执行转移类指令导致PC值改变

# 数据寻址

什么是数据寻址？确定本条指令的地址码指明的真实地址

- 形式地址 A （address）
- 有效地址 EA（effective address）
- 指令格式：操作码（OP）+ 寻址特征（何种寻址方式）+形式地址

## 十大寻址方式

- 直接寻址
  - 定义：
    - 形式地址即为操作数的真实地址
  - 优点：
    - 简单，指令执行阶段仅访问一次主存，不需要专门计算操作数的地址
  - 缺点：
    - 形式地址位数决定了指令操作数的寻址范围，操作数地址不易修改
- 间接寻址
  - 定义：
    - 形式地址不是操作数的真实地址，而是操作数有效地址的所在存储单元的地址（即是地址的地址）
  - 优点：
    - 可扩大寻址范围（有效地址EA的位数大于形式地址A的位数）
    - 便于编制程序（用间接寻址可以方便地完成子程序的返回）
  - 缺点：
    - 指令在执行阶段要多次访问主存（一次间接寻址需要两次访存，多次寻址需要根据存储字的最高位确定几次访存）
- 寄存器寻址
  - 定义：
    - 在指令字中直接给出操作数所在的寄存器编号，即是：EA=Ri，其操作数在由Ri所指的寄存器内
  - 优点：
    - 指令在执行阶段不访问主存，只访问寄存器
    - 指令字短且执行速度快，支持向量/矩阵运算
  - 缺点：
    - 寄存器价格昂贵，计算中寄存器个数有限
- 寄存器间接寻址
  - 定义：
    - 寄存器Ri中给出的不是一个操作数，而是操作数所在主存单元的地址，即是：EA=(Ri)
  - 特点：
    - 与一般的间接寻址相比速度更快，但指令的执行阶段需要访问主存（寄存器中存的是操作数的主存地址，真实的操作数在主存中）
- 隐含寻址
  - 定义：
    - 不是明显地给出操作数的地址，而是在指令中隐含着操作数的地址
  - 优点：有利于缩短指令字长
  - 缺点：需增加存储操作数或隐含地址的硬件
- 立即寻址
  - 定义：
    - 形式地址A就是操作数本身，又称为立即数，一般采用补码形式。#表示立即寻址特征。执行指令不用访问主存。
  - 优点：
    - 执行指令阶段不访问主存，指令执行时间最短
  - 缺点：形式地址A的位数为n，且立即数采用补码时，可表示的数据范围为：-2^(n-1)~2^(n-1)-1
- 基址寻址
  - 定义：
    - 以程序的起始存放地址作为“起点”;
    - BR：基址寄存器，全程：base address register
      - 注：基址寄存器是面向操作系统的，其内容由操作系统或管理程序确定。在程序执行 过程中，基址寄存器的内容不变（作为基地址），形式地址可变（作为偏移量）。 当采用通用寄存器作为基址寄存器时，可由用户决定哪个寄存器作为基址寄存器，
        但其内容仍由操作系统确定。 优点：可扩大寻址范围（基址寄存器的位数大于形式地址A的位数）；用户不必考虑自 己的程序存于主存的哪一空间区域，故有利于多道程序设计，以及可用于编制浮动程序 （整个程序在内存里边的浮动） 。
    - 将CPU中基址寄存器（BR）的内容加上指令格式中的形式地址A，从而形成操作数的有效地址，即是：EA = (BR) + A
- 相对寻址
  - 定义：
    - 以程序计数器PC所指地址作为“起点”
- 变址寻址
  - 定义：
    - 程序员自己决定从哪里作为“起点”
- 堆栈寻址
  - 定义：操作数存放在堆栈中，隐含使用堆栈指针（sp）作为操作数地址。
  - SP：stack pointer
  - 堆栈：存储器（或专门寄存器组）中一块特定的按“先进先出”原则管理的存储区。该存储区中被读/写的地址是用一个专门的寄存器给出的，该寄存器成为堆栈指针（SP）

  - 划分：
    - 硬堆栈：用寄存器实现
    - 软堆栈：用主存实现
